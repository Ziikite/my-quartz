# Smart Composer - Obsidian 안에서 AI 글쓰기와 PKM을 연결하는 플러그인

## 한 줄 요약

Smart Composer는 기존에 Cursor와 Obsidian을 오가며 하던 AI 기반 노트 작성·수정·자동완성 경험을 Obsidian 내부에서 더 간편하게 구현하게 해주는 플러그인으로 소개된다.

## 핵심 내용

### 1. 문제의식: Cursor + Obsidian 왕복의 번거로움

작성자는 원래 `Cursor + Obsidian`을 오가며 사용했다고 말한다. Cursor는 마크다운과 AI 편집에 강하지만, Obsidian Vault를 외부 에디터로 열어야 하므로 앱 전환 비용이 생긴다.

댓글에서도 작성자는 아직 Cursor + Obsidian 조합이 손에 익어 Cursor를 더 많이 쓰지만, Smart Composer는 “스위칭이 덜해서” 좋다고 평가한다.

### 2. 제안 도구: Smart Composer

Smart Composer는 Obsidian 안에서 AI 기능을 사용할 수 있게 하는 플러그인으로 언급된다.

원문에서 강조된 장점:
- Obsidian + AI 사용 가능
- Cursor와 Obsidian을 오가는 것보다 간편함
- Obsidian 파일을 Cursor로 불러오지 않아도 비슷한 경험 가능
- 개발 실력이 좋은 팀이 만든 것으로 보인다는 댓글 평가

### 3. Cursor를 메모 도구로 쓰는 이유

댓글에서 작성자는 Cursor를 메모·글쓰기 도구로 쓰는 장점을 구체적으로 설명한다.

- 빈 페이지에서 시작하기 어려울 때 구조를 짜기 쉽다.
- Cursor 안에서 AI로 바로 수정할 수 있다.
- Notion AI처럼 자동 생성 기능을 사용할 수 있다.
- 문장 작성 시 자동완성 기능을 사용할 수 있다.
- 텍스트도 결국 “수정 대상”이기 때문에 코드 에디터의 기능과 잘 맞는다.
- Git으로 버전 관리가 가능하다.
- Cursor의 웹 검색 기능을 통해 추가 리서치도 가능할 수 있다.

### 4. Smart Composer의 잠재적 장점

Smart Composer의 강점은 Cursor의 AI 글쓰기 경험을 Obsidian 내부로 가져오는 데 있다.

- 앱 전환 감소
- Obsidian Vault 맥락 유지
- 노트 작성과 AI 편집의 결합
- Cursor를 별도로 열지 않고도 유사한 워크플로우 가능
- PKM, 글쓰기, 리서치, 회고 노트 작성에 활용 가능성

### 5. 비용과 구독 피로도

작성자는 ChatGPT, Claude, Cursor를 이미 구독하고 있는데 Smart Composer에 API를 연결하면 추가 과금이 발생하는 점을 아쉬움으로 언급한다.

즉, 이 도구의 도입 여부는 단순 기능 비교가 아니라 다음 두 요소의 균형 문제다.

- 스위칭 비용 감소
- AI API 추가 과금 또는 “AI 월세” 증가

## 주요 관찰 포인트

### 1. Obsidian 안으로 AI가 들어오는 흐름

이 스레드는 AI 지식관리의 흐름이 “Obsidian을 Cursor 같은 AI 에디터에서 여는 방식”에서 “Obsidian 자체 안에 AI 플러그인을 붙이는 방식”으로 이동하고 있음을 보여준다.

### 2. Cursor의 본질은 코드 에디터가 아니라 텍스트 편집 에이전트일 수 있다

작성자가 말하듯 코딩도 결국 텍스트 수정이다. 그렇다면 Cursor의 AI 편집 기능은 코드뿐 아니라 글쓰기, 회고, 노트 정리에도 자연스럽게 확장된다.

### 3. 앱 전환 비용은 생각보다 큰 생산성 병목이다

Cursor와 Obsidian 조합이 강력해도, 두 앱을 오가는 과정은 흐름을 끊는다. Smart Composer의 가치는 기능 자체보다 “작업 맥락을 유지한 채 AI를 호출한다”는 데 있을 수 있다.

### 4. AI 도구의 진짜 비용은 월 구독료 + API 비용 + 전환 비용이다

Smart Composer가 유용하더라도 이미 ChatGPT, Claude, Cursor를 구독 중이라면 추가 API 과금이 부담이 된다. AI 도구 선택에서는 기능뿐 아니라 구독 포트폴리오 관리가 필요하다.

### 5. PKM 도구 선택 기준이 변하고 있다

과거 노트앱 선택 기준이 블록 구조, 폴더 구조, 디자인, 링크 기능이었다면, 이제는 AI 친화성, 로컬 마크다운 접근성, 자동완성, 검색·리서치 연동, 버전 관리까지 포함된다.

## 활용 예시

### 예시 1. Obsidian에서 글 아웃라인 만들기

- 빈 노트에 주제만 적는다.
- Smart Composer로 글의 목차와 섹션 구조를 생성한다.
- 생성된 구조를 직접 수정한다.
- 관련 노트 링크를 추가해 글쓰기 재료로 삼는다.

### 예시 2. 회고 노트 확장하기

- 하루 일지를 짧게 적는다.
- Smart Composer에게 다음 질문으로 회고를 확장하게 한다.
  - 오늘 가장 중요했던 사건은 무엇인가?
  - 감정적으로 반복되는 패턴은 무엇인가?
  - 내일 바로 실행할 액션은 무엇인가?
- 결과를 주간 리뷰 노트에 연결한다.

### 예시 3. 독서 노트 정리하기

- Readwise 등으로 들어온 하이라이트를 Obsidian에 저장한다.
- Smart Composer로 핵심 주장, 인상 깊은 문장, 내 생각, 적용 아이디어를 분리한다.
- 관련 주제 노트와 `[[위키링크]]`로 연결한다.

### 예시 4. Cursor와 Smart Composer 역할 나누기

- Obsidian + Smart Composer:
  - 짧은 노트 작성
  - 회고
  - 하이라이트 정리
  - 빠른 문장 보완
- Cursor + Obsidian Vault:
  - 여러 파일을 넘나드는 대규모 정리
  - 폴더 단위 리팩터링
  - Git 기반 버전 관리
  - 긴 글 초안의 구조적 편집

## 내 Obsidian에 적용해볼 체크리스트

- [ ] Smart Composer 플러그인이 현재 Obsidian 버전에서 안정적으로 동작하는지 확인한다.
- [ ] 어떤 LLM API를 연결해야 하는지 확인한다.
- [ ] 이미 구독 중인 ChatGPT, Claude, Cursor와 비용이 중복되는지 계산한다.
- [ ] 짧은 노트 작성은 Smart Composer, Vault 전체 작업은 Cursor처럼 역할을 나눌지 정한다.
- [ ] 자주 쓰는 프롬프트를 Obsidian 템플릿이나 플러그인 설정에 저장한다.
- [ ] AI가 수정한 노트는 원문을 덮어쓰기보다 diff 또는 백업 방식으로 관리한다.

## 바로 써볼 프롬프트 예시

### 노트 구조화

```text
이 노트를 Obsidian에서 다시 찾고 활용하기 쉽게 정리해줘.
- 핵심 요약
- 주요 주장
- 내 생각을 덧붙일 질문
- 관련 키워드
- 연결하면 좋을 [[위키링크]] 후보
형식으로 정리해줘.
원문 표현은 가능한 한 보존하고, 새로 추가한 해석은 별도로 표시해줘.
```

### 회고 확장

```text
아래 일지를 자기성장 관점에서 확장해줘.
- 오늘의 핵심 사건
- 감정 패턴
- 배운 점
- 반복되는 문제
- 내일 실행할 작은 행동 1개
로 정리해줘.
과하게 조언하지 말고, 내가 쓴 표현을 최대한 살려줘.
```

### 글쓰기 보조

```text
아래 아웃라인을 글 초안으로 확장해줘.
각 섹션마다 2~3문단을 작성하고,
내 기존 표현을 최대한 유지해줘.
부족한 논리나 추가하면 좋은 예시는 따로 '보완 제안'에 적어줘.
```

## 한계와 주의점

- Smart Composer가 API 기반이라면 사용량에 따라 추가 비용이 발생할 수 있다.
- Obsidian 안에서 바로 수정되는 AI 도구는 원문 훼손 위험이 있으므로 백업이나 Git 관리가 있으면 좋다.
- 플러그인 안정성과 개인정보 처리 방식을 확인해야 한다.
- 짧은 노트에는 편하지만, Vault 전체를 대상으로 한 대규모 작업은 Cursor가 더 나을 수 있다.
- AI 자동완성에 익숙해지면 내 문체와 사고 흐름이 평준화될 수 있다.

## 다음 액션

- Smart Composer 설치 가능 여부와 설정 방식을 확인한다.
- 테스트용 노트 하나를 만들어 요약, 확장, 문체 수정, 위키링크 추천을 실험한다.
- Cursor와 Smart Composer의 역할을 나누는 기준을 정한다.
- 비용이 부담된다면 기존 Cursor 워크플로우를 유지하되, Obsidian 내부 작업이 꼭 필요한 경우만 Smart Composer를 사용한다.
