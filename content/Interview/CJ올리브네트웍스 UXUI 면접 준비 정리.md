# CJ올리브네트웍스 UXUI 면접 준비 정리

## 한 줄 요약

이번 면접 준비의 핵심은 **“다양한 도메인에서 사용자 맥락을 데이터와 리서치로 읽고, 비즈니스에 적용 가능한 UX 솔루션으로 구조화하는 사람”**이라는 메시지를 일관되게 전달하는 것이다.

## 핵심 포지셔닝

### 나를 설명하는 중심 문장

> 저는 다양한 도메인에서 사용자 맥락을 리서치와 데이터로 분석하고, 이를 실제 서비스 구조와 UX 솔루션으로 연결해 온 UX/UI 기획자입니다. CJ올리브네트웍스에서는 내부 계열사와 외부 고객사의 문제를 “처음 보는 사용자의 눈”으로 진단하고, 실행 가능한 UX 컨설팅 결과물로 만드는 사람이 되고 싶습니다.

### 면접에서 반복해서 가져갈 키워드

- 사용자 맥락
- 데이터 기반 인사이트
- UX 컨설팅
- 대내외 프로젝트
- 다양한 도메인 적응력
- 문제 정의 → 솔루션 설계 → 실행/검증
- 이해관계자 조율
- 현장 맥락 기반 UX
- 비즈니스 임팩트와 사용자 가치의 균형

## 지원 서류 핵심 정리

### 1. 지원동기 / 성장계획

핵심 메시지:
- CJ올리브네트웍스는 CJ ONE, One PG, OneOrder 등 일상과 연결된 디지털 솔루션을 제공한다.
- 사용자의 여정과 데이터를 기반으로 통합 경험을 설계한다는 점이 지원자의 경험과 맞닿아 있다.
- 입사 후에는 다양한 도메인의 사용자 맥락을 분석하고 UX 솔루션을 제안하는 전문가로 성장하고 싶다.

강조할 연결점:
- GLN에서 의료비 결제와 등록금 결제라는 서로 다른 도메인을 하나의 UX 기준으로 통합 설계한 경험
- B2C, 어드민, 물류, 대외 고객사 등 다양한 맥락을 빠르게 파악하는 역량
- 단기: 실무 도메인 이해와 UX 기획 실무 강화
- 장기: 사용자 문제를 선제적으로 정의하고 UX 솔루션을 제안하는 컨설턴트형 UX 전문가

면접용 답변 클로징:

> 결국 제가 CJ올리브네트웍스에서 하고 싶은 일은 단순히 화면을 개선하는 것이 아니라, 다양한 도메인의 사용자 맥락을 읽고 서비스 구조 자체를 더 자연스럽게 만드는 일입니다. 이 점에서 제가 해온 리서치 기반 UX 기획 경험을 대내외 UX 컨설팅 업무에 바로 연결할 수 있다고 생각합니다.

### 2. 핵심 직무 역량

핵심 역량:
- 리서치로 사용자 맥락을 파악하는 능력
- 복잡한 기술/서비스를 사용자의 언어로 재해석하는 능력
- 분석 결과를 실제 UX 설계와 비즈니스 성과로 연결하는 능력

대표 경험:

#### 한국GM 음성 UX 가이드라인

- 문제: 음성 안내가 반복되거나 맥락 없이 제공될 때 사용자가 혼란을 느끼고 무시함
- 행동:
  - 유저 섀도잉과 인터뷰 수행
  - 운전 여정 단계별로 정보 구조화
  - 상황 맥락이 담긴 설명형 발화로 변경
  - 38명 A/B 테스트, 18명 인터뷰 수행
- 결과:
  - 안전감 12% 향상
  - 편안함 14% 향상
  - 지속사용의도 9% 향상
  - 전사 가이드라인 채택

#### 티맥스클라우드 마이그레이션 신사업

- 문제: 클라우드 마이그레이션은 기술적 진입장벽과 심리적 부담이 큼
- 행동:
  - 구글 검색 결과와 경쟁사 IA 분석
  - “복잡함, 어려움, 유지보수” 등 심리적 장벽 키워드 도출
  - 코드 기반 환경 분석을 설문 기반 환경 진단으로 전환 제안
  - Grafana를 벤치마킹한 대시보드 설계
- 결과:
  - PoC 승인
  - 신사업 채택

### 3. 어려움 / 실패 극복 경험

대표 경험:
- 티맥스클라우드 CLAS 서비스 개선

문제:
- 성과 지표 없이 계열사별 웹페이지가 개별 운영됨
- 자사 기술 디펜던시로 성과 수집이 어려움
- 계열사별 UI, 용어, 정책이 달라 사용자 경험이 불일치

행동:
- 샘플 페이지 제작
- GA와 Beusable 적용
- 유입 경로와 이벤트 데이터 분석
- UI 간소화 및 CTA 재배치
- Tableau 대시보드 제작
- UI, 용어, 정책, 서비스 플로우를 포함한 통합 매뉴얼 작성
- 계열사 실무진과 공통 목표 합의

결과:
- 상품 확인 버튼 클릭률 30% 향상
- CLAS 서비스 전반을 팀에서 리딩
- 외부 행사에서 통합 서비스 공식 공개

면접용 클로징:

> 이 경험을 통해 UX 기획은 단순히 화면을 개선하는 일이 아니라, 사용자 행동 데이터와 이해관계자의 목표를 함께 읽고 서비스 운영 기준을 정렬하는 일이라는 것을 배웠습니다.

### 작년 면접 총평

잘한 점:
- 자기소개 흐름은 자연스러웠다.
- 수치 근거가 있어 설득력이 있었다.
- 경력 서술은 연대기적으로 명확했다.
- 협업 갈등 해결 답변이 강점으로 보였다.
- 역질문 수준이 좋았고 호기심이 드러났다.

아쉬운 점:
- VUI × 인클루시브 디자인 연결이 약했다.
- 트렌드 답변에서 CJ올리브네트웍스 업무와의 연결이 부족했다.
- “나를 한마디로” 답변이 직무 능력에만 치우쳤다.
- 답변이 장황하고 `~했고요`, `이제`, `아무래도`가 반복됐다.
- 일부 답변에서 마무리 정리 멘트가 없었다.

## 이번 면접에서 반드시 보완할 약점

### 1. VUI × 인클루시브 디자인

면접관이 듣고 싶었던 핵심:

> VUI는 GUI 접근이 어려운 사용자에게 중요한 대체 접점이 될 수 있으며, 시각·운동 능력의 제약을 줄여 접근성을 확장하는 인클루시브 디자인의 핵심 채널이다.

모범 답변 구조:

1. VUI 정의
2. 인클루시브 디자인에서의 포지션
3. 본인 경험 연결
4. 마무리 한 문장

답변 예시:

> 제가 생각하는 VUI는 음성이라는 모달리티를 통해 사용자의 의도와 상황 맥락을 해석하고, 그에 맞는 응답을 설계하는 인터랙션 시스템입니다. 인클루시브 디자인 관점에서 VUI가 중요한 이유는 GUI가 기본적으로 시각과 손 조작을 전제로 하는 반면, VUI는 그 전제를 낮출 수 있기 때문입니다. 예를 들어 저시력 사용자, 손 떨림이 있는 고령자, 운전 중 두 손을 사용하기 어려운 사용자에게 음성은 더 자연스러운 접근 채널이 될 수 있습니다. 한국GM 프로젝트에서도 주행 중 인지 부하가 높은 상황을 고려해 음성 안내를 여정별로 재구성했는데, 이 역시 특정 사용자를 위한 보조 기능을 넘어 모든 사용자의 접근 가능성을 넓히는 설계였다고 생각합니다. 결국 VUI는 보조 기술이 아니라 사용자 접근성을 확장하는 주요 UX 채널이라고 생각합니다.

### 2. 트렌드 / AI 툴 답변

보완 방향:
- 좋아하는 앱이나 트렌드 소개로 끝내지 않는다.
- 반드시 CJ올리브네트웍스 업무와 연결한다.

클로징 예시:

> 이처럼 사용자가 의식하지 않아도 행동 변화를 유도하는 UX 설계는, 어드민이나 B2B 서비스에서도 사용자의 워크플로우 부담을 줄이는 방향으로 응용할 수 있다고 생각합니다.

### 3. 나를 한마디로 표현하기

기존 키워드:
- 지연PT

보완 방향:
- 직무 능력뿐 아니라 “사람 김지연”의 성향을 함께 보여준다.

답변 예시:

> 저는 주변에서 ‘지연PT’라고 불립니다. 자료를 조사할 때 최소 30건 이상 찾아보고, 복잡한 내용을 한 페이지로 구조화해 전달하는 습관이 있기 때문입니다. 그런데 이건 단순히 자료를 잘 찾는다는 의미만은 아닙니다. 친구들이 고민을 이야기할 때도 바로 해결책을 말하기보다, 먼저 그 사람이 진짜 필요로 하는 것이 무엇인지 정리해주는 편입니다. 그래서 주변에서 ‘얘기하기 편하다’는 말을 자주 듣습니다. CJ올리브네트웍스에서도 다양한 이해관계자의 의견을 구조화하고, 팀이 같은 방향으로 움직일 수 있게 돕는 지연PT가 되고 싶습니다.

### 4. 말 습관 개선

줄여야 할 표현:
- 이제
- 아무래도
- ~했고, ~했고요
- 길게 이어지는 설명

연습 규칙:
- 답변은 `결론 → 근거 → 사례 → 마무리` 순서로 말한다.
- 답변 하나에 핵심 메시지는 1개만 둔다.
- 마지막 문장은 반드시 `그래서 ~라고 생각합니다` 형태로 닫는다.
- 녹음 후 `이제`, `아무래도`, `했고요` 횟수를 세고 답변당 1회 이하로 줄인다.

## 예상 질문별 답변 뼈대

### Q1. 자기소개

핵심 구조:
- 데이터와 인사이트로 맞춤형 니즈를 파악하는 UX 기획자
- 다양한 도메인 경험: 모빌리티, 클라우드, 핀테크, 커머스
- 대표 성과: 한국GM VUI, GLN FAQ, 결제 서비스, 티맥스 CTA 30% 향상
- CJ올리브네트웍스의 다양한 프로젝트에서 사용자 맥락 기반 UX를 설계하고 싶음

짧은 답변:

> 안녕하세요. 데이터와 인사이트로 사용자의 니즈를 구조화하고 UX 솔루션으로 연결해 온 김지연입니다. 저는 모빌리티, 클라우드, 핀테크 등 다양한 도메인에서 리서치 기반 UX 기획을 수행했습니다. 한국GM 프로젝트에서는 음성 UX 가이드라인을 설계해 안전감과 편안함을 개선했고, 티맥스클라우드에서는 GA와 Beusable 데이터를 바탕으로 상품 확인 버튼 클릭률을 30% 향상시켰습니다. 최근에는 GLN에서 외국인 대상 의료비·등록금 결제 서비스와 FAQ 개선을 기획했습니다. 이러한 경험을 바탕으로 CJ올리브네트웍스에서도 다양한 도메인의 사용자 문제를 정의하고 실행 가능한 UX 솔루션으로 연결하는 기획자가 되고 싶습니다.

### Q2. CJ올리브네트웍스 지원동기

핵심 구조:
- CJ올리브네트웍스는 생활문화와 디지털 기술이 만나는 회사
- CJ ONE, One PG, OneOrder 등 통합 경험을 설계하는 기업
- 본인은 사용자 데이터와 맥락을 기반으로 서비스 구조를 설계해왔다
- 대내외 UX 컨설팅과 다양한 도메인이 본인의 성장 방향과 맞다

### Q3. 퇴사사유 + 이직사유 + 지원동기

답변 예시:

> 전 직장은 계약기간 종료라는 현실적인 이유도 있었지만, 그 과정에서 제 커리어 방향이 더 분명해졌습니다. UX 기획과 개선 업무를 경험하면서 사용자 데이터를 분석하고 인사이트를 도출해 전략 방향을 세우는 과정이 가장 흥미롭고 잘 맞는다는 것을 알게 되었습니다. 특히 고객의 문제를 깊이 이해하고 상황별로 적합한 솔루션을 제공하는 UX 전문가로 성장하고 싶어졌습니다. CJ올리브네트웍스는 다양한 도메인에서 대내외 고객의 문제를 해결하는 UX 컨설팅 업무를 수행한다는 점에서 제가 지향하는 방향과 맞아 지원했습니다.

### Q4. 입사 후 어떤 도메인의 일을 하고 싶은가?

답변 예시:

> 저는 장기적으로 외부 고객사 대상 UX 컨설팅, 즉 대외사업 도메인에서 일하고 싶습니다. GLN에서 크로스보더 결제 UX를 설계할 때 가장 어려웠던 것은 기술보다도 이 서비스를 처음 보는 사람의 눈으로 다시 보는 일이었습니다. 외부 고객사 UX 컨설팅 역시 매번 새로운 도메인의 서비스를 처음 보는 눈으로 진단하고, 문제를 정의해 개선 방향을 제시하는 일이라고 생각합니다. 다만 입사 초기에는 CJ 계열사 내부 프로젝트를 통해 회사의 서비스 설계 기준과 일하는 방식을 먼저 익히고, 그 바탕 위에서 외부 고객사의 문제를 CJ의 관점으로 풀어내는 컨설턴트형 UX 기획자로 성장하고 싶습니다.

### Q5. 하고잡이 정신과 본인의 결

핵심 답변:
- 하고잡이 = 주어진 조건을 탓하지 않고 가진 것으로 최선의 결과를 만드는 태도
- 석사 과정에서 예산 부족, 노후 장비, 적은 피험자 수에도 연구를 진행한 경험
- 직접 장비를 수리하고, 정량 리서치와 인터뷰를 병행해 데이터 밀도를 높임
- CJ올리브네트웍스의 다양한 제약 상황에서도 방법을 찾는 태도로 기여 가능

### Q6. 협업 갈등 / 이해관계자 조율

추천 사례:
- 티맥스 CLAS 통합 매뉴얼
- 계열사별 다른 UI, 용어, 정책을 통합해야 했던 상황
- 공통 목표를 “일관된 서비스 경험”으로 재정의
- 구글 시트와 컨플루언스로 매뉴얼화
- 결과적으로 서비스 전반을 팀에서 리딩

답변 구조:
- 상황: 계열사별 기준이 달라 사용자 혼란 발생
- 과제: 통합 기준 마련과 이해관계자 합의
- 행동: 데이터 근거 제시, 공통 목표 설정, 매뉴얼 제작
- 결과: 일관된 UX 확보, 팀 리딩 범위 확대
- 배움: 협업은 의견을 맞추는 것이 아니라 판단 기준을 공유하는 것

### Q7. 본인이 들어오면 어떤 역량을 발휘할 수 있는가?

답변 방향:
- 다양한 도메인에 빠르게 적응하는 리서치 역량
- 사용자의 언어로 복잡한 기술을 풀어내는 UX 기획 역량
- 데이터와 인터뷰를 결합해 문제를 정의하는 역량
- 이해관계자를 정렬하는 문서화/구조화 역량

### Q8. 임원과의 커뮤니케이션

답변 방향:
- 임원에게는 디테일보다는 의사결정에 필요한 핵심 구조를 먼저 전달
- 문제, 근거, 선택지, 기대효과를 명확히 제시
- 한국GM 임원진 회의에서 VUI 가이드라인을 긍정적으로 평가받은 경험 연결

### Q9. 다른 회사에 떨어진 경험 / 왜 떨어진 것 같은지

답변 방향:
- 방어적으로 말하지 않는다.
- 부족했던 점을 구체적으로 인정한다.
- 보완 행동을 말한다.

예시:

> 이전 면접에서는 제가 가진 직무 경험을 충분히 설명하려다 보니, 답변이 다소 길어지고 사람으로서의 저를 보여주는 답변이 부족했다고 느꼈습니다. 특히 VUI와 인클루시브 디자인처럼 개념을 연결하는 질문에서 마무리 정리가 약했습니다. 이후에는 답변을 결론, 근거, 사례, 마무리 구조로 정리해 연습하고 있고, 직무 역량뿐 아니라 제가 어떤 방식으로 생각하고 협업하는 사람인지도 함께 전달하려고 준비하고 있습니다.

## 프로젝트별 답변 카드

### 1. 한국GM 음성 UX 가이드라인

- 문제: 음성 UX 사용률 저조, 맥락 없는 반복 안내로 사용자 피로 발생
- 리서치: 섀도잉, 인터뷰, 토픽모델링, A/B 테스트
- 인사이트: 사용자는 음성 기능 자체보다 “왜 지금 이 안내가 필요한지”를 알고 싶어함
- 설계: 운전 여정 단계별 정보 제공, 설명형 발화 구조
- 성과: 안전감 12%, 편안함 14%, 지속사용의도 9% 향상, 전사 가이드라인 채택
- 연결: VUI는 운전 중 인지 부하를 줄이고 접근성을 높이는 인클루시브 UX 채널

### 2. 한국GM 전기차 충전 UX 전략

- 문제: 충전 중 OTT 소비 가설이 실제 사용자 니즈와 다름
- 리서치: 스토리보드 이미지, 센터페시아 목업, 사용자 인터뷰
- 인사이트: 사용자는 콘텐츠 소비보다 충전 불확실성, 지연, 피로를 더 크게 느낌
- 설계: 충전 예약 UI, 휴식 가능한 라운지형 충전소, 브랜드 프로모션 공간
- 연결: 사용자가 말한 표면 니즈보다 실제 맥락 속 페인포인트를 재정의한 경험

### 3. 티맥스클라우드 PreSales 서비스 페이지 개선

- 문제: 계열사별 웹페이지 운영으로 일관성 부족, 성과 측정 체계 부재
- 리서치/분석: GA, Beusable, 유입 경로, 이벤트 데이터 분석
- 인사이트: 화면 탐색 단계가 많아 사용자가 복잡하게 느끼고 이탈
- 설계: UI 간소화, CTA 재배치, 콘텐츠 레이아웃 개선, Tableau 대시보드 구축
- 성과: 상품 확인 버튼 클릭률 30% 향상
- 연결: B2B에서도 사용자 행동 데이터 기반 UX 개선이 가능함

### 4. 티맥스클라우드 클라우드 마이그레이션

- 문제: 기존 사용자를 직접 찾기 어렵고, 기술 진입장벽이 높음
- 리서치: 구글 검색 결과 크롤링, 경쟁사 IA 분석
- 인사이트: “복잡함, 어려움, 유지보수”가 핵심 심리 장벽
- 설계: 코드 기반 분석을 설문 기반 환경 진단으로 전환, 현황 대시보드 설계
- 성과: PoC 승인, 신사업 채택
- 연결: 복잡한 기술을 사용자가 이해 가능한 UX로 번역한 경험

### 5. GLN 신사업 서비스 기획

- 문제: 외국인 대상 의료비/등록금 결제 서비스의 실제 사용자 접근이 어려움
- 리서치: 중국·몽골 의료 관광객 조사, 병원 코디네이터 인터뷰
- 인사이트: 의료비 결제는 상담 직후 빠르게 완료되어야 하므로 중간 마찰이 없어야 함
- 설계: 비가입 결제 서비스, MediPay GLN 기획 및 디자인
- 성과: 개발·기획 랩업 세션을 통한 얼라인 확보
- 연결: 직접 사용자 접근이 어려운 B2B/B2B2C 환경에서 대리 사용자 리서치를 활용한 경험

### 6. GLN FAQ 개선

- 문제: FAQ 콘텐츠가 충분해도 사용자가 자신의 상황을 찾지 못해 문의 발생
- 리서치: CS 데이터 3,929건 분석, 운영부 인터뷰
- 인사이트: 카테고리와 문장 구조가 사용자의 상황 인식에 맞지 않음
- 설계: 토픽모델링 기반 FAQ 재분류, UX 라이팅 원칙 구체화
- 결과: 간단하고, 명료하고, 친근하고, 행동하기 쉬운 콘텐츠로 개선
- 연결: 콘텐츠 UX도 사용자 문제 해결의 중요한 접점

### 7. GLN MediPay / EduPay 통합 설계

- 문제: 의료비 결제와 등록금 납부는 유사한 결제 서비스처럼 보이지만 핵심 니즈와 상태값이 다름
- 인사이트:
  - MediPay: 빠른 결제 완료가 중요
  - EduPay: 납부 현황 파악이 중요
- 해결:
  - 플로우와 컴포넌트는 공통 기준으로 통합
  - 상태값은 서비스별로 분기 처리
- 배움:
  - 통합 설계에서 중요한 것은 무엇을 같게 할지가 아니라, 무엇을 다르게 유지해야 하는지 판단하는 것
- CJ 연결:
  - 다양한 도메인을 다루는 CJ올리브네트웍스에서도 이 기준을 적용 가능

- 직무 이해를 넓히는 질문
- 입사 후 기대 역할을 확인하는 질문

### 추천 질문

1. 제가 UX/UI 직무로 합류하게 된다면, 팀에서 신입 또는 주니어에게 가장 기대하는 역할은 무엇인지 궁금합니다.
2. CJ올리브네트웍스의 UX 컨설팅 업무에서 내부 계열사 프로젝트와 외부 고객사 프로젝트는 문제 정의 방식이나 협업 방식에서 어떤 차이가 있는지 궁금합니다.
3. 다양한 도메인을 다루는 과정에서 UX 기획자가 가장 먼저 익혀야 하는 회사 내부의 기준이나 일하는 방식은 무엇인지 궁금합니다.
4. 최근 CJ올리브네트웍스에서 UX 관점이 특히 중요해지고 있는 도메인이나 프로젝트 유형이 있다면 어떤 것인지 궁금합니다.
