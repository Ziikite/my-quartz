# AI는 UI 문제를 얼마나 신뢰성 있게 찾아낼까?

Jim Lewis, PhD · Jeff Sauro, PhD · Will Schiavone, PhD · Lucas Plabst, PhD  
2026년 4월 28일

AI는 영상을 “볼” 수 있는 것처럼 보인다. 그리고 AI가 영상을 볼 수 있다면, UI 문제도 추출할 수 있을 가능성이 있다. 이는 AI가 UX 리서치를 지원할 잠재력이 있음을 시사한다.

그렇다면 AI가 영상을 “보고” 몇 가지 문제를 감지할 수는 있을 것이다. 하지만 같은 영상을 AI에 여러 번 넣으면 매번 같은 결과가 나올까?

신뢰성은 중요하다. AI가 실행할 때마다 다른 결과를 내놓는다면, 그 추론이 아무리 그럴듯해 보여도 신뢰하기 어렵다.

AI의 신뢰성을 평가할 때 영향을 줄 수 있는 변수는 많다.

- AI 모델: Gemini, ChatGPT, Claude, Grok 등
- 버전: 모델은 매달 바뀌고 있다.
- temperature 같은 LLM 설정: 출력의 무작위성에 영향을 준다.
- 프롬프트: 무엇을 묻는지, 심지어 몇 번 묻는지도 영향을 준다.

고려할 변수가 많지만, 어딘가에서 시작해야 한다. 이 글에서는 AI의 문제 감지 신뢰성을 평가하기 위한 첫 단계로, 두 개의 인기 AI 챗봇이 같은 영상에서 사용성 문제를 얼마나 일관되게 식별하는지 살펴보았다.

## 연구 설정

두 LLM, 즉 ChatGPT-5.4 Thinking과 Gemini 3 Flash Thinking에게 영상을 검토하고 발견한 사용성 이슈를 나열하게 했다. 각 LLM은 네 번씩 실행하여 LLM 내부의 일관성을 확인했고, 기본 설정만 사용했다. 두 모델 모두 MeasuringU가 유료 “프로” 구독으로 사용하는 범용 LLM이었다. 영상 1은 전체 6분 영상 중 처음 15초 클립이다.

**영상 1:** 참가자가 OpenTable 웹사이트에서 스시 레스토랑을 검색하는 첫 15초.

영상 하단에 표시된 과제는 OpenTable.com에서 다음을 수행하는 것이었다.

> “소리 내어 생각하며 진행해 주세요. 내일 오후 5시 이후 아무 시간에 덴버, 콜로라도의 스시 레스토랑에서 네 명을 위한 예약을 하세요. 선택하는 레스토랑이 가장 낮은 가격대나 가장 높은 가격대가 아니어야 합니다. 이 기준을 충족하는 레스토랑 중 전체 평점, 고객 리뷰, 사진을 살펴보고 가장 마음에 드는 곳을 선택하세요. 개인정보나 계정 정보를 요청받기 전까지 예약 과정을 가능한 한 진행하세요. 예약을 완전히 확정하지는 마세요. 레스토랑 이름과 예약 시간을 적어 두세요. 과제 후 이 정보에 대해 질문을 받을 것입니다.”

사용한 프롬프트는 다음과 같았다.

> “사용성 테스트 중 진행자는 참가자가 웹사이트, 모바일 앱, 소프트웨어 프로그램 등을 탐색하며 과제를 수행하는 동안 참가자의 행동을 추적해야 합니다. 참가자들이 스시 레스토랑에서 테이블을 예약하도록 요청받은 사용성 테스트 영상을 봐 주시기 바랍니다. 영상을 보는 동안 참가자가 과제를 완료하려고 시도하면서 겪는 문제를 찾아 주세요. 예를 들어 사용자가 이동한 경로를 기록하고, 사용자가 마주친 이슈와 웹사이트의 어떤 부분이 문제를 유발했을 수 있는지 설명할 수 있습니다. 이 지시를 이해했다면 알려 주세요. 그러면 검토할 영상을 드래그해 넣겠습니다. 영상을 볼 준비가 되었나요?”

이 질문에 대한 LLM의 응답은 항상 “예”에 해당하는 형태였다.

이 연구에서는 AI의 종류만 달리했다. ChatGPT와 Gemini를 비교했으며, 영상, 프롬프트, LLM 버전과 설정은 일정하게 유지했다. 향후 연구에서는 이러한 변수들도 변경해 볼 계획이다.

## 신뢰성 평가

AI에게 요청하면 AI는 무엇이든 산출한다. 각 실행마다 AI 모델이 “발견한” 사용성 문제 목록을 수집했다.

예를 들어 영상에서, 그리고 현재 OpenTable 웹사이트에서도 눈에 띄는 문제는 검색 필드에 “Denver”를 입력하면 이전에 선택했던 cuisine, 즉 sushi가 제거된다는 점이다. 이는 필터와 검색 경험을 서툴고 번거롭게 만든다.

문제 발견의 신뢰성, 즉 일관성을 평가하기 위해 ChatGPT와 Gemini 간, 그리고 각 모델 내부에서 **any-2 agreement**를 계산했다. 여기서는 모델들을 평가자처럼 다루었다.

any-2 agreement는 UX 맥락에 맞춘 **Jaccard similarity coefficient, J**의 한 형태다. 두 이진 측정값의 교집합을 합집합으로 나눈 비율이다. 평가자가 두 명 이상일 때 전체 any-2 agreement는 각 평가자 쌍의 any-2 agreement 평균이다.

### any-2 agreement 계산하기

Y와 C가 각각 독립적으로 사용성 이슈 목록을 만들었다고 하자. Y의 목록에는 14개 이슈, C의 목록에는 17개 이슈가 있고, 두 목록에 공통으로 포함된 이슈가 10개라면 any-2 agreement는 교집합 10개를 합집합 21개, 즉 14 + 17 - 10으로 나눈 값이다. 결과는 48%, 즉 10/21이다.

평가자 효과는 잘 문서화되어 있기 때문에, UX 연구자들 사이에서 완벽한 합의를 기대하지는 않는다. 이런 통제된 연구, 즉 평가자들이 같은 참가자가 같은 과제를 수행하는 장면을 보는 상황에서 여러 인간 평가자 간 일반적인 any-2 agreement에 대한 저자들의 최선의 추정치는 12개 평가를 기준으로 47%다. 연구가 통제되지 않은 경우 예상 any-2 agreement는 약 27%다.

따라서 any-2 agreement가 일반적인 수준인지, 대략 50%, 낮은 수준인지, 대략 25%, 높은 수준인지, 대략 75%를 판단하기 위한 대략적 기준이 생긴다.

## 그룹 내부 결과

분석의 첫 단계는 각 “평가자” 그룹, 즉 ChatGPT와 Gemini에 대해 평균 any-2 agreement를 계산하여 그룹 내부 신뢰성 수준을 추정하는 것이었다.

### ChatGPT의 신뢰성은 보통 수준이었다

아래는 ChatGPT 네 번 실행의 결합 문제 목록이다.

- 1. 복잡한 검색 필드, placeholder 텍스트, 예상치 못한 동작 때문에 사용자가 크게 지연되었다. 사용자는 검색창 드롭다운에서 sushi를 선택했지만, Denver가 아니라 기본값인 Dallas 기준으로 선택했다. 실행 1, 2, 3에서 발견.
- 2. 검색 필드에 “Denver”를 입력하자 이전에 선택한 sushi cuisine이 사라졌다. 실행 1, 2, 3에서 발견.
- 3. 필터가 도움이 되지 않았다. 실행 2에서 발견.
- 4. 86개 cuisine을 훑는 것은 부담스럽고, 그 목록에 sushi가 없다는 점이 문제를 더했다. 실행 3에서 발견.
- 5. sushi를 검색 필드에 입력했을 때 현재 위치가 사라지지 않은 점에 사용자가 놀랐다. 실행 2에서 발견.
- 6. sushi 검색 결과에 sushi가 아닌 레스토랑이 많이 포함되었다. 실행 1, 4에서 발견.
- 7. 검색 결과에서 cuisine 정보가 약하게 제시되었다. 실행 1, 3, 4에서 발견.
- 8. 참가자가 가격대 필터를 놓친 것으로 보였다. 평점 기준으로 정렬한 뒤 설명에서 가격대를 확인했다. 실행 1에서 발견.
- 9. 최고 평점 기준 정렬이 많은 비-sushi 레스토랑을 목록 상단에 올렸다. 실행 4에서 발견.
- 10. UI가 좋은 의사결정 지원 없이 브라우징을 밀어붙인다. 실행 1, 2, 3에서 발견.
- 11. 선택한 결과가 sushi가 아니라 seafood로 표시되었다. 실행 1, 3, 4에서 발견.
- 12. 참가자가 예약 양식에 도달하지 못했기 때문에 과제가 완료되지 않았다. 실행 2에서 발견.

ChatGPT 실행 쌍별 any-2 agreement는 다음과 같았다.

- Run 1 vs Run 2: 30%
- Run 1 vs Run 3: 63%
- Run 1 vs Run 4: 38%
- Run 2 vs Run 3: 33%
- Run 2 vs Run 4: 0%
- Run 3 vs Run 4: 25%

**전체 any-2 agreement가 31%**였기 때문에 ChatGPT 평가의 신뢰성은 **보통, fair** 수준이었다. 네 번의 실행 모두에서 식별된 문제는 없었고, 12개 중 5개 문제만 세 번의 실행에서 식별되었다. 실행 2와 실행 4는 공통 문제가 전혀 없었다.

### Gemini의 신뢰성은 더 좋았다

아래는 Gemini 네 번 실행의 결합 문제 목록이다.

- 1. 검색 필드에 “Denver”를 입력하자 이전에 선택한 sushi cuisine이 사라졌다. 실행 1, 2, 3, 4 모두에서 발견.
- 2. 86개 cuisine을 훑는 것은 부담스럽고, 그 목록에 sushi가 없다는 점이 문제를 더했다. 실행 1, 2, 3, 4 모두에서 발견.
- 3. 참가자가 페이지에서 “sushi”를 찾기 위해 Ctrl-F를 사용했지만 찾지 못했다. 실행 1, 2, 3, 4 모두에서 발견.
- 4. 참가자가 가격대 필터를 놓친 것으로 보였다. 평점 기준으로 정렬하고 설명에서 가격대를 살폈다. 실행 1, 3, 4에서 발견.
- 5. 참가자가 가장 높은 가격대를 선택했다. 실행 2에서 발견.
- 6. 참가자가 낮은 평점순으로 정렬하고 싶어 했지만 해당 옵션이 없었다. 실행 3에서 발견.
- 7. 좌석 옵션은 시간을 선택한 뒤에만 제시되었다. 실행 1에서 발견.
- 8. 시간을 5:10으로 설정했다. 실행 2에서 발견.
- 9. 선택한 결과가 sushi가 아니라 seafood로 표시되었다. 실행 2에서 발견.

Gemini 실행 쌍별 any-2 agreement는 다음과 같았다.

- Run 1 vs Run 2: 38%
- Run 1 vs Run 3: 67%
- Run 1 vs Run 4: 80%
- Run 2 vs Run 3: 38%
- Run 2 vs Run 4: 43%
- Run 3 vs Run 4: 80%

**전체 any-2 agreement가 57%**였기 때문에 Gemini 평가의 신뢰성은 **좋은, good** 수준이었다. 9개 문제 중 3개가 네 번의 실행 모두에서 식별되었고, 9개 중 4개가 최소 세 번의 실행에서 식별되었다.

## 그룹 간 결과

분석의 두 번째 단계는 LLM 간 평균 any-2 agreement를 계산하여 그룹 간 신뢰성을 추정하는 것이었다.

ChatGPT와 Gemini 실행 간 any-2 agreement는 다음과 같았다.

- GPT 1 vs Gem 1: 40%
- GPT 1 vs Gem 2: 40%
- GPT 1 vs Gem 3: 33%
- GPT 1 vs Gem 4: 40%
- GPT 2 vs Gem 1: 20%
- GPT 2 vs Gem 2: 20%
- GPT 2 vs Gem 3: 17%
- GPT 2 vs Gem 4: 20%
- GPT 3 vs Gem 1: 40%
- GPT 3 vs Gem 2: 75%
- GPT 3 vs Gem 3: 33%
- GPT 3 vs Gem 4: 40%
- GPT 4 vs Gem 1: 0%
- GPT 4 vs Gem 2: 33%
- GPT 4 vs Gem 3: 0%
- GPT 4 vs Gem 4: 0%

**전체 any-2 agreement가 28%**였기 때문에 AI 간 신뢰성은 **낮은, low** 수준이었다. 이는 50%보다 25%에 가까운 값이다.

## 요약과 논의

저자들은 UX 리서처 커뮤니티와 마찬가지로 AI가 업무를 지원할 수 있는 역할에 큰 관심을 가지고 있다. 참가자가 과제를 완료하려고 시도하는 장면을 관찰하는 것은 UX 리서치의 기본이지만 노동집약적인 활동이다. 따라서 AI 지원이 조금이라도 부담을 줄여준다면 환영할 만하다.

이 연구는 영상에서 사용성 문제를 찾는 ChatGPT-5.4 Thinking과 Gemini 3 Flash Thinking의 능력을 조사하기 위한 첫 단계였다. 단일 영상에 대해 각 LLM을 네 번씩 실행하여 사용성 문제 목록 형태의 평가를 수집했다.

이 글에서는 각 평가 그룹 내부, 즉 ChatGPT와 Gemini 내부의 any-2 agreement와 AI 간 any-2 agreement를 평가했다. 핵심 발견은 다음과 같다.

### Gemini는 좋은 신뢰성을 보였고, ChatGPT는 보통 수준이었다

ChatGPT의 평균 any-2 agreement는 31%였다. 저자들은 서로 다른 평가자, 서로 다른 방법, 또는 서로 다른 사용자를 비교할 때 이런 수준의 신뢰성을 예상한다고 설명한다. 바람직한 수준보다 낮은 것은 분명하지만, 업계에서 받아들일 수 있는 수준으로 간주되는 범위에는 들어간다.

Gemini의 평균 any-2 agreement는 57%로 좋았다. 문헌과 저자들의 인간 평가자 연구를 기준으로 보면, 57%는 평균 47%보다 높고 수용 가능한 범위의 상단에 가까운 값이다.

### Gemini와 ChatGPT 사이의 그룹 간 신뢰성은 낮았다

ChatGPT와 Gemini 사이의 any-2 agreement는 28%로 낮았다. 이는 같은 영상을 서로 다른 사람이 검토할 때의 평균보다 약 20포인트 낮다. 좋은 결과라고 보기는 어렵다.

### 신뢰성은 정확성이 아니다

LLM이 식별한 문제가 인간 평가자가 발견한 문제만큼 관련성이 있는가? 이 질문에는 아직 답하지 않았다. 향후 분석에서 다룰 예정이다. 하지만 정확성, 즉 타당성을 갖기 위해서는 먼저 일관된, 즉 신뢰할 수 있는 결과를 확립해야 한다. 적어도 이 영상과 프롬프트에서는 Gemini의 성능이 충분히 신뢰할 만했다.

### 인간 대 AI 비교는 곧 다룰 예정이다

저자들은 분석을 이제 막 시작했다. 향후 글에서는 이 LLM들과 전문 인간 UX 연구자들이 생성한 문제 목록 사이의 any-2 agreement를 비교할 예정이다.
