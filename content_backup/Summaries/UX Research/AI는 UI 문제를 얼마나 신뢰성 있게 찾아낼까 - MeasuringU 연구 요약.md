# AI는 UI 문제를 얼마나 신뢰성 있게 찾아낼까 - MeasuringU 연구 요약

원문 번역 노트: [[2026-04-28 How Reliable Is AI at Finding UI Problems - 한국어 번역]]

## 한 줄 요약

AI가 사용성 테스트 영상을 보고 UI 문제를 찾아낼 수는 있지만, 같은 영상과 같은 프롬프트에서도 모델별·실행별 일관성이 크게 달라지므로 “발견 결과”를 그대로 신뢰하기보다는 신뢰성 검증 절차와 인간 검토를 함께 설계해야 한다.

## 핵심 결론

- **Gemini 3 Flash Thinking은 비교적 안정적이었다.** 같은 영상에 대해 4회 실행했을 때 평균 any-2 agreement가 57%였다.
- **ChatGPT-5.4 Thinking은 보통 수준의 일관성을 보였다.** 4회 실행 평균 any-2 agreement는 31%였다.
- **두 모델 간 결과 일치는 낮았다.** ChatGPT와 Gemini 간 평균 any-2 agreement는 28%로, 같은 영상을 인간 평가자들이 보는 통제 연구의 평균치인 약 47%보다 낮았다.
- **신뢰성은 정확성이 아니다.** 같은 답을 반복해서 내는 것과 실제로 중요한 문제를 맞게 찾는 것은 다르다. 이 연구는 정확성보다 먼저 “반복 실행 시 얼마나 일관적인가”를 확인한 것이다.

## 연구 구조

### 대상 과제

참가자는 OpenTable에서 다음 조건의 예약을 진행해야 했다.

- Denver, CO의 스시 레스토랑
- 내일 오후 5시 이후
- 4명 예약
- 최저가·최고가가 아닌 가격대
- 평점, 리뷰, 사진을 살펴보고 가장 마음에 드는 곳 선택
- 개인정보나 계정 정보를 요구받기 전까지 진행하되 최종 예약 확정은 하지 않음

### AI 평가 조건

- 모델: ChatGPT-5.4 Thinking, Gemini 3 Flash Thinking
- 각 모델 4회 실행
- 같은 영상, 같은 프롬프트, 같은 모델 버전/기본 설정 사용
- AI에게 “사용성 테스트 영상을 보며 참가자가 과제를 수행하는 동안 겪는 문제와 원인을 기록하라”고 요청

### 평가 지표

**any-2 agreement**를 사용했다. 이는 두 평가자가 발견한 문제 목록의 교집합을 합집합으로 나눈 값이다.

- 예: A가 14개, B가 17개 문제를 발견했고 공통 문제가 10개라면 합집합은 21개이므로 agreement는 10/21 = 48%.
- 인간 UX 평가자 간 통제 연구의 평균 기준: 약 47%.
- 통제되지 않은 연구의 평균 기준: 약 27%.
- 대략적 해석:
  - 25% 부근: 낮음
  - 50% 부근: 일반적/수용 가능
  - 75% 부근: 높음

## 관찰과 인사이트

### 1. “AI가 본다”보다 “AI가 일관되게 본다”가 더 중요하다

AI가 영상에서 문제를 하나라도 찾는 것은 이제 놀라운 일이 아닐 수 있다. 하지만 리서치 업무에 쓰려면 같은 입력에 대해 유사한 결과를 반복적으로 낼 수 있어야 한다. 이 글의 핵심은 AI의 가능성보다 **반복 가능성**을 측정했다는 데 있다.

### 2. 같은 모델도 실행마다 문제 목록이 달라진다

ChatGPT는 4회 실행 중 모든 실행에서 공통으로 발견한 문제가 없었다. 특히 Run 2와 Run 4는 공통 문제가 0%였다. 이는 AI 분석 결과를 한 번만 받아서 “전체 문제 목록”으로 취급하는 방식이 위험할 수 있음을 보여준다.

### 3. 모델 간 차이는 실무에서 더 큰 문제다

Gemini 내부 신뢰성은 57%로 괜찮았지만, ChatGPT와 Gemini 사이의 agreement는 28%였다. 즉 어떤 AI를 쓰느냐에 따라 리서치 결과물이 달라질 수 있다. 조직에서 AI 기반 UXR 워크플로우를 운영한다면 모델 변경, 버전 업데이트, 프롬프트 변경이 결과의 연속성을 깨뜨릴 수 있다.

### 4. AI의 사용성 문제 발견은 “보조 평가자”로 보는 편이 안전하다

이 연구만 보면 AI를 독립 리서처로 보기보다는 후보 문제를 넓게 제안하는 보조 평가자로 사용하는 것이 더 적절하다. 특히 Gemini처럼 내부 일관성이 더 높은 모델도, 인간 전문가와의 정확성 비교가 끝난 것은 아니다.

### 5. 평가 지표를 UXR 문맥에 맞춰야 한다

단순히 “좋은 답변 같아 보인다”가 아니라 any-2 agreement 같은 지표를 사용하면 AI 리서치 결과물을 더 객관적으로 검토할 수 있다. AI 기반 분석 파이프라인을 도입할 때도 모델 성능을 UX 리서치 관점의 신뢰성 지표로 관리할 필요가 있다.

## 실무 활용 예시

### AI로 사용성 테스트 영상을 1차 분석할 때

- 같은 영상을 한 번만 분석하지 말고 최소 2~3회 반복 실행한다.
- 반복 실행에서 공통으로 나온 문제와 한 번만 나온 문제를 구분한다.
- 공통 문제는 “우선 검토 후보”로, 단발성 문제는 “추가 확인 후보”로 분류한다.
- 모델별 결과가 다를 수 있으므로, 중요한 연구에서는 두 개 이상의 모델을 비교한다.

### AI 분석 결과를 리서처가 검토할 때

- AI가 발견한 문제를 그대로 인사이트로 쓰지 않는다.
- 각 문제에 대해 실제 영상 증거, 타임스탬프, 참가자 발화, UI 상태를 확인한다.
- AI가 놓친 문제를 찾기 위해 인간 리서처의 독립 리뷰를 병행한다.
- AI가 제안한 원인 설명과 실제 인터랙션 증거를 분리해서 검토한다.

### 조직 내 AI UXR 품질관리 기준으로

- 모델명과 버전을 기록한다.
- 프롬프트를 고정하고 변경 이력을 남긴다.
- temperature 등 설정값을 기록한다.
- 동일 샘플 영상으로 정기적인 회귀 테스트를 수행한다.
- 모델 업데이트 후 기존 기준 영상에 대한 agreement 변화를 확인한다.

## 체크리스트

AI로 사용성 문제를 분석하기 전에:

- [ ] 분석할 영상과 과제 설명이 명확한가?
- [ ] 참가자의 목표, 제약 조건, 성공 기준이 프롬프트에 포함되어 있는가?
- [ ] 모델명, 버전, 설정값을 기록했는가?
- [ ] 같은 입력에 대해 반복 실행할 계획이 있는가?
- [ ] 결과를 비교할 기준, 예: any-2 agreement 또는 인간 평가자 목록이 있는가?

AI 분석 결과를 받은 뒤:

- [ ] 반복 실행 간 공통 문제를 표시했는가?
- [ ] 모델 간 불일치 항목을 따로 모았는가?
- [ ] 각 문제에 영상 증거와 타임스탬프를 연결했는가?
- [ ] AI의 문제 식별과 원인 해석을 분리했는가?
- [ ] 인간 리서처가 최종 판단했는가?

## 한계와 주의점

- 이 연구는 단일 영상과 단일 과제를 기반으로 한다.
- ChatGPT와 Gemini의 특정 버전, 특정 설정, 특정 프롬프트에 대한 결과다.
- 신뢰성만 평가했으며 정확성, 즉 인간 전문가가 발견한 실제 문제와 얼마나 일치하는지는 아직 평가하지 않았다.
- AI가 더 일관적으로 문제를 찾는다고 해서 더 정확하거나 더 유용하다는 뜻은 아니다.
- 모델은 계속 업데이트되므로 같은 실험도 시간이 지나면 결과가 달라질 수 있다.

## 내 작업에 적용할 만한 프롬프트 패턴

```text
당신은 UX 리서처의 보조 평가자입니다. 아래 사용성 테스트 영상/전사/관찰 로그를 보고 참가자가 과제를 수행하는 동안 겪은 사용성 문제를 찾아 주세요.

각 문제는 다음 형식으로 작성하세요.
1. 문제 요약
2. 관찰 근거 또는 타임스탬프
3. 사용자의 목표 달성에 미친 영향
4. 가능한 UI 원인
5. 심각도 후보: 낮음/중간/높음
6. 확신도: 낮음/중간/높음

주의:
- 영상에서 직접 관찰 가능한 내용과 추론을 분리하세요.
- 확실하지 않은 항목은 “추가 확인 필요”로 표시하세요.
- 문제를 과장하지 말고, 참가자 행동 근거를 반드시 연결하세요.
```
