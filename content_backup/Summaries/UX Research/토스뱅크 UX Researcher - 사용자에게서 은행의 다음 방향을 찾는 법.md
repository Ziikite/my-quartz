# 토스뱅크 UX Researcher - 사용자에게서 은행의 다음 방향을 찾는 법

원문 노트: [[2026-02-25 사용자에게서 은행의 다음 방향을 찾는 UX Researcher - 원문]]

## 한 줄 요약

토스뱅크의 UX Researcher는 사용자 인터뷰나 리포트 작성에 머무르지 않고, 금융 맥락에서 사용자의 문제를 구조화해 제품 방향·비즈니스 전략·조직의 사용자 중심 의사결정 문화까지 연결하는 역할을 한다.

## 핵심 메시지

- **리서치는 보고서가 아니라 의사결정 재료다.** 사용자의 니즈와 페인포인트를 발굴하고, 은행 전반의 관점에서 고객과 소통할 수 있는 비즈니스 인사이트로 전환한다.
- **도메인이 달라도 사용자 이해 역량은 전이된다.** 신호철 리서처는 자동차·모빌리티에서 금융으로 이동했지만, 사용자를 깊이 이해하고 제품·사업으로 연결하는 본질적 역량을 핵심 자산으로 보았다.
- **토스뱅크의 리서치는 전략 실행과 연결된다.** 사회초년생을 주거래 고객으로 확보하기 위한 리서치는 문제 정의, 설계, 분석, 결론, 활용 판단까지 이어졌고, 이후 스쿼드의 OKR 액션 아이템으로 연결되었다.
- **금융 UX의 핵심은 공급자 중심 관행을 사용자 관점에서 다시 묻는 것이다.** “은행은 원래 이렇다”가 아니라 “사용자는 은행에 무엇을 기대하고 필요로 하는가?”를 조직이 자연스럽게 묻게 만드는 일이 중요하다.
- **시니어 리서처의 기여는 리서치 양보다 판단력이다.** 지금 조직에 정말 필요한 리서치가 무엇인지 판단하고, 결과를 통해 조직이 의미 있는 방향으로 움직이게 하는 것이 핵심이다.

## 아티클 구조

### 1. 토스뱅크 UX Researcher의 역할

신호철 리서처는 금융 맥락에서 사용자를 입체적으로 이해하고, 이를 바탕으로 제품과 비즈니스의 방향을 제안한다. 또한 구성원들이 업무 전반에서 사용자를 중심에 둘 수 있도록 돕는 퍼실리테이터 역할도 맡는다.

구체적으로는:

- 사용자를 이해하는 방법을 내부 세션으로 공유한다.
- 사용자가 토스뱅크에서 좋은 경험을 하고 있는지 진단한다.
- 제품과 비즈니스 방향을 사용자 인사이트 기반으로 제안한다.
- 구성원들이 사용자 중심으로 사고하고 의사결정하도록 돕는다.

### 2. 도메인 전환의 기준

금융권 경험이 없다는 점은 고민이었지만, 핵심은 특정 산업 지식보다 사용자를 이해하고 그것을 제품으로 연결하는 힘이었다. 자동차·모빌리티에서 IVI UX 기획, 신사업 전략, Product Manager 역할을 경험한 경력은 금융에서도 의미 있는 기반이 되었다.

이 대목은 UX 리서처의 커리어 전환에서 중요한 관점을 준다.

- 도메인 지식은 새로 배울 수 있다.
- 사용자 문제를 정의하고 구조화하는 역량은 전이 가능하다.
- 인사이트를 제품·사업 의사결정으로 연결한 경험은 산업을 넘어 가치가 있다.

### 3. 토스뱅크 조직 문화의 특징

아티클에서 강조되는 토스뱅크의 문화는 다음과 같다.

- 사용자 중심 접근에 대한 공감대
- 현실적 제약 속에서도 대안을 찾으려는 태도
- 성취와 실패를 공유하는 케이스 스터디 문화
- 명확한 DRI(Directly Responsible Individual)
- 의사결정권자가 DRI를 존중하며 의견을 나누는 구조

이는 리서처가 인사이트를 내도 의사결정 과정에서 무뎌지는 환경과 대비된다. 리서치가 실제 제품 완성도로 연결되려면, 인사이트의 품질뿐 아니라 조직의 의사결정 구조도 중요하다는 점을 보여준다.

### 4. 임팩트 사례: 사회초년생 주거래 고객 리서치

입사 후 첫 과제로 사회초년생을 토스뱅크의 주거래 고객으로 확보하기 위한 리서치를 진행했다.

이 프로젝트의 특징:

- 주어진 질문에 답하는 리서치가 아니었다.
- 문제 정의부터 리서치 설계·실행·분석·결론 도출·활용 판단까지 전 과정을 주도했다.
- 사회초년생의 주거래은행 활동을 이해하기 위한 프레임워크를 만들었다.
- 계좌 사용 방식, 자금 흐름 관리, 금융 상품 선택 등 복합적인 행동을 의사결정에 활용 가능한 구조로 정리했다.
- 결과는 신설 스쿼드의 OKR 달성을 위한 액션 아이템으로 이어졌다.

이 사례는 UX Researcher가 “사용자 세그먼트 이해 → 전략적 관점 도출 → 조직 실행 과제 연결”까지 맡을 수 있음을 보여준다.

## 관찰과 인사이트

### 리서처는 사용자와 조직 사이의 번역자다

토스뱅크 UX Researcher는 사용자의 목소리를 듣는 데서 멈추지 않는다. 사용자의 행동과 맥락을 조직이 이해하고 실행할 수 있는 언어로 번역한다. 이때 리서치 결과는 보고서가 아니라 제품팀·비즈니스팀·스쿼드가 움직일 수 있는 판단 근거가 된다.

### 금융 도메인에서는 “입체적인 사용자” 이해가 중요하다

금융 사용자는 때로 비합리적으로, 때로 매우 이성적으로 의사결정한다. 금융 행동은 계좌 사용, 자금 흐름, 상품 선택, 신뢰, 리스크 인식, 생활 맥락이 복합적으로 얽혀 있다. 따라서 단일 행동이나 표면적 니즈만 보는 리서치로는 충분하지 않다.

### 사용자 중심 문화도 설계 대상이다

아티클에서 UX Research팀의 핵심 과제 중 하나는 토스뱅크 안에 사용자 중심 사고와 의사결정 문화를 정착시키는 것이다. 이는 리서치팀의 역할이 프로젝트 단위 분석을 넘어 조직 역량 구축까지 확장된다는 뜻이다.

### 새로운 도메인의 낯섦은 질문의 자산이 될 수 있다

낯선 도메인에 들어가면 기존 구성원에게 당연한 것을 다시 물을 수 있다. “은행은 원래 이렇다”는 전제를 사용자 관점에서 다시 정의하는 질문은 도메인 외부자의 강점이 될 수 있다.

## 실무 활용 예시

### UX 리서치 프로젝트를 전략과 연결할 때

1. 사용자 세그먼트를 명확히 정의한다.
2. 사용자의 핵심 행동 프레임워크를 만든다.
3. 행동, 니즈, 페인포인트, 의사결정 맥락을 구조화한다.
4. 제품·비즈니스 관점에서 어떤 방향성이 필요한지 제안한다.
5. 스쿼드의 OKR 또는 액션 아이템으로 연결한다.
6. 리서치 결과가 실행됐는지 후속 추적한다.

### 금융 UX 리서치에서 볼 질문

- 사용자는 금융 활동에서 본질적으로 어떤 문제를 해결하려 하는가?
- 현재 사용자는 어떤 우회 방법이나 기존 관행에 적응하고 있는가?
- 사용자는 어떤 순간에 합리적으로, 어떤 순간에 감정적으로 판단하는가?
- 공급자 중심 구조 때문에 사용자가 감수하는 불편은 무엇인가?
- 사용자가 은행에 기대하지만 명시적으로 말하지 않는 것은 무엇인가?
- 이 인사이트를 제품 전략, 커뮤니케이션, 지표로 어떻게 연결할 수 있는가?

### 리서처 커리어 전환을 고민할 때

- “나는 어떤 도메인을 알고 있는가?”보다 “나는 문제를 어떻게 정의하는가?”를 점검한다.
- 사용자 이해를 제품·사업 결과로 연결한 사례를 정리한다.
- 낯선 도메인에서 던질 수 있는 질문을 강점으로 활용한다.
- 개인 리서치 산출물보다 팀 미션과 조직 임팩트를 우선한다.
- 역할을 좁게 정하기보다 리서치를 출발점으로 제품·사업·조직에 어떻게 기여할지 열어둔다.

## 체크리스트

리서치가 실행 가능한 전략으로 이어지고 있는지 점검하기:

- [ ] 리서치 질문이 제품 또는 비즈니스 의사결정과 연결되어 있는가?
- [ ] 사용자 행동을 설명하는 프레임워크가 있는가?
- [ ] 단순 니즈 목록이 아니라 행동 맥락과 의사결정 구조를 설명하는가?
- [ ] 결과가 스쿼드의 OKR, 로드맵, 액션 아이템과 연결되는가?
- [ ] 사용자 관점과 현실적 제약이 충돌할 때 대안을 제안하는가?
- [ ] 리서치 결과가 조직 구성원의 사용자 이해 역량을 높이는 데 쓰이는가?
- [ ] 리서치가 끝난 뒤 실제 실행과 임팩트를 추적하는가?

## 내 작업에 적용할 포인트

- UX 리서치 결과물을 만들 때 “인사이트 → 전략 방향 → 실행 과제”의 연결을 명시한다.
- 사용자 세그먼트 리서치에서는 행동 프레임워크를 먼저 세우고, 그 프레임워크가 의사결정에 어떻게 쓰이는지까지 설계한다.
- 리서치 보고서에는 발견점만 쓰지 말고, 조직이 어떤 질문을 다시 해야 하는지 포함한다.
- 도메인 전환이나 커리어 포트폴리오를 정리할 때는 산업 지식보다 문제 정의력, 사용자 이해력, 제품·사업 연결력을 중심으로 서술한다.

## 한계와 주의점

- 이 글은 토스 커리어 아티클이므로 채용 브랜딩 맥락을 포함한다.
- 토스뱅크 UX Researcher의 전체 업무 범위라기보다는 인터뷰이 신호철님의 경험과 관점을 중심으로 정리된 사례다.
- 조직 문화 설명은 내부 구성원의 인터뷰 기반이므로, 외부에서 그대로 일반화하기보다는 리서치 조직 설계의 참고 사례로 보는 것이 좋다.
